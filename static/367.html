1.1 Network Hardware

1.1.1 Types of Transmission

  • Point-to-point
    Single sender, single receiver. The packet may be passed along many sources before it reaches its destination. Telephone lines.
  • Broadcasting
    When the communication channel is shared by all computers on the network. packets are received by every computer, and have a field in the packet designating the intended receiver. wifi/ethernet fit in this category.
  • Multicasting
    Like broadcasting, but only transmitting to a subset of the machines on the network.
  • Packet
    'short' block of data sent/received by network. Size of the packet is hardware-defined.

1.1.2 Types of Networks

  • Personal Area Network (PAN)
    Networks that are very short-ranged (within reach usually.) Bluetooth is a good example, which creates a very simple master-slave network in which all peripheral bluetooth devices connect to the master (computer, car, etc.)
  • Wide Area Network (WAN)
    WANs cover large geographic regions (countries even) and work very similarly to LANs or MANs, just on a larger scale. In a WAN, there will be several hosts (computers running user programs), and the communication subnet. The subnet is a collection of transmission lines and switches (routers) the move data between the hosts.
  • Virtual Private Networks (VPN)
    Many individual networks at different site joined together to create the effect of one large network that includes all of the users in each network

1.1.3 Types of Connections

  • Connection-Oriented Services
    Sockets and telephones work in this way. You first open a connection, use it for some time, and finally close it when you're done. Data transmission is done through a type of pipe, where the sender pushes data in to the pipe, and the receiver pulls it out.
  • Connectionless Services
    Rather than having a designated sender/receiver, all messages using this type of service have the absolute address of the recipient. Messages are routed through connected nodes in the system until they reach their destination.

1.1.4 Internetwork

The connection of two or more networks. Internet is the best-known internetwork.

1.1.5 Physical Medium

Peers across different machines will not directly share data. Instead, they will pass information to the layer below it until it reaches the bottom, the physical medium. Here is where the actual transmission occurs.

1.1.6 Statistical Multiplexing

Networks provide a service to the connected hosts with their available resources such as bandwidth. To be efficient, the network must have some way to divide the resources in a way that doesn't interfere with other hosts. Sharing the resources dynamically is called is statistical multiplexing based on demand.

1.2 Flow Control

When fast-sending connections swamp a very slow receiver, flow control is used to provide feedback to the origin of the transmission. It is like a warning to slow dow

1.3 Protocol

Agreement between communicating parties on how communcations should proceed.

1.4 Protocol Stack

A list of protocols used by each layer in a certain system, one protocol per layer.

1.5 Peers

Entities that are communicating using some protocol. When a network is comprised of multiple layers, peers would be the software or hardware on the same layer that communicate using some protocol.

1.6 Network Architecture

Simply the set of layers that comprise a network. A specification should include enough information for an implementer to write the program/build hardware for each layer so it will follow the specified protocol.

1.7 Service vs. Protocol

Simply put, a service is a set of primitives (operations) that it makes available to the layer above it. It is a definition of the operation it is prepared to make on behalf of the layer above.

A protocol is a set of rules that govern the format and meaning of the messages/packets that are exchanged between peer entities on a layer.

Entities use protocols to implement services. For this reason protocols can change to any degree, as long as the primitives the service provides do not change.

1.8 Types of Services

1.8.1 Datagram Service

This service is unreliable because there is no acknowledgement that a packet was received. Usually there is a high probability that the message will be received but no guarantee.

1.8.2 Acknowledged Datagram

When an established connection isn't necessary, you can use the datagram service but request a "receipt" that reassures the sender that message was definitely received.

1.8.3 Request-Reply Service

A client sends a single message to the server, and it responds with a single message response.

1.9 Open Systems Interconnection (OSI)

This is a standardized model whos structure is important, but the actual protocols are no longer used. It has seven layers which were designed to separate well-defined functions while keeping the number of layers manageable.

1.9.1 Physical Layer

Transmits raw bits over a communication channel. Its purpose is to make sure each bit sent as a 0 is received as a 0, the implementation is in charge of handling the mechanical/electrical aspect of the transportation.

1.9.2 Data Link Layer

This layer is charged with transforming raw transmission of data in to an apparently error-free line. It should hide the error-handling implementation from the layer above, and only allow correct packets to bubble up.

To accomplish this by having the sender break up the input data in to data frames (some arbitrary number of bytes) and transmit these frames sequentially. When the service needs to be reliable, the receiver must confirm by replying with an acknowledgment frame.

1.9.3 Network Layer

Controlling the operations of the subnet is the responsibility of the network layer. A key issue is determining how packets are routed, which path they should take from a source to its destination. Routes can be hardcoded in a static table, dynamically calculated for each connection, or some medium between.

Congestion can be caused when there are too many packets in the subnet, and the network layer's job is to make sure this doesn't happen. In general, the Quality of Service (QoS) is an issue of the network layer.

Finally, sending packets between multiple networks can also be problematic; the receiving network may use a different protocol, have packet length limits, etc. The network layer should handle these possible errors and correct them seamlessly.

1.9.4 Transport Layer

The transport layer communicates directly with the connected entity. It should take data from the layer above, break it up (if needed), and then send it to the network layer. Furthermore, it should make sure that all the data was sent/received correctly in a way that shouldn't be noticeable to the layers above it. The communication is not impeded by routers imbetween like layers below, it should have a direct line to the destination machine.

1.9.5 Session Layer

A session offers various services like dialog control (as in conversation, whos turn it is to talk), token management (don't let multiple parties attempt the same critical operation at the same time), and synchronization (allowing long transmissions to be picked up where they left off in the event of a crash or disconnect).

1.9.6 Presentation Layer

While other layers are concerned with moving bits around, this layer dicates the syntax and semantics of that information. This is important for computers that have different data represntations. An abstract and standard encoding can be used and interpreted by any peer that knows the protocol.

1.9.7 Application Layer

Here is the final layer that contains protocols that are needed and used by actual users. For example the HTTP protocol would be applicable, as it is the highest-level protocol used in the stack.

2 Physical Layer

Information transmitted by varying some physical property. In books, the ink varies; in wires we can vary voltage, current, etc., fiberoptics vary lights.

2.1 Bandwidth

Hardware has an affect on different frequencies, and the width of that frequency range is called bandwidth. Square waves have a lot of higher harmonics, but a small bandwidth removes a lot of harmonics.

2.2 Noise

Random variations in a signal that wasn't caused by the source.

2.3 Baseband

Signals from 0 to max frequency

2.4 Passband

Signals shifted occupy a higher frequency

2.5 Guided Transmission Media

2.5.1 Twisted Pair

Usually two thin copper wires twisted to make give them similar noise. If they were separated, radiation would create different amounts of noise in each wire.

  • Simplex
    One transmitter sends data across the twisted pair to the receiver. Essentially just a one way wire.
  • Half-Duplex
    One twisted pair, where the transmitter/receiver switch roles, so its one way at a time, but the direction can change.
  • Full-Duplex
    Two pairs, concurrent data in both directions.

2.5.2 Coaxial Cable

Cable TV wires, used in original ethernet.

2.5.3 Power Lines

Actually possible to transmit data across power lines, high bandwidth but high voltage (dangerous).

2.5.4 Fiber Optics

Long haul transmission (long distance?) Different modes, single/multi where it would be one-way or two-way. In single mode, you can send 100Gbps over 100km.

2.5.5 Wireless

Uses the radio spectrum (10**4Hz to 10**16Hz) of frequencies.

2.6 Modulation

A change in signal to encode information (modulation). Extracting information from a signal (demodulation). You can do both analog and digital modulation. Modulation takes the carrier wave (medium of transmission) and bends it to represent the data that needs to be sent.

2.6.1 Amplitude Modulation

Change the amplitude of the wave to match a sine wave (for analog) or for bit data (digital).

2.6.2 Frequency Modulation

Modifying the frequency to be interpreted as a sine wave or bit data. For bit data you could have two frequencies, one to represent 0 and one for 1.

3 Data Link Layer

3.1 Transmission Errors

Drop bits, add bits, change bits. This layer should effectively hide these from the network layer.

3.2 Frames

Chunks that the bitstream is broken down in to. Some arbitrary agreed upon size.

3.2.1 Establishing Frames

  • Character Count
    Breaks the message up in to sequences of bytes, prepending a byte that notes the total bytes in the frame (including byte count). An error in the byte count throws off the whole frame structure (ie. 5 -> 24).
  • Start/End Characters
    Uses "byte stuffing". In ASCII, there are special characters DLE (data link escape), SOH (start of header), and EOT(end of transmission). Each frame has [DLE][SOH] … [DLE][SOH].
    • Problems
      Binary data can accidentally trigger an end of frame by having a DLE bit pattern. To remedy this, "byte stuffing" is used. When transmitting binary data, if it sees a DLE bit pattern, it "stuffs" a second one next to it. Then when the receiver parses the data, it can remove the extras.
  • Start/End Bit Patterns
    Uses "bit stuffing". Frames start with 0x7E, when it sees 5 '1' bits in a row, add a '0'.

3.3 Types of Service

3.3.1 Unacknowledged, connectionless

No attempt to detect or correct errors. Can be used effectively over a very reliable medium for large data where errors do not matter much.

3.3.2 Acknowledged, connectionless

Each frame is acknowledged by the receiver. When the receiving data link layer reads a layer, it can ask for a retransmit

3.3.3 Connection Service

Sender, receiver establish a connection. Frames are numbered, and transmitted in order.

3.4 Error Control

3.4.1 Provide sender with feedback

Positive/negative acknowledgement after receiving a frame.

3.4.2 Frame never reaches a frame

Sender has a timeout period, if hit hasn't had an acknowledgement at the end of the timeout, it will resend the frame. Can result in mutliple copies if the receiver was just too slow to respond.

  • Detection
    Add redundancy to the data.
    • Parity
      First bit says whether there are even/odd bits in the byte.
    • Checksums
      -#- Learn for test.

3.4.3 Correction

Upon detecting an error, it can fix the error on its own without a retransmission.

3.5 Protocols

3.5.1 Simple Protocol (stop and wait)

Data –> … <– ACK No ACK back, assumes that it didn't get there. Retransmit.

3.5.2 Sliding Window

  • Window - buffer
    Fixed size of bytes/packets. Can send all these packets without an ACK. Sender retains copies of the packets until receives ACK, will resend those that it didn't receive. When the ACKs have been received it can discard the packets. "ACK n" specifies that they received ALL packets up to packet n.

3.9 Ethernet

3.9.1 Frame (in bytes)

preamble - 8 dest address - 6 source address - 6 type - 2 data - 0 to 1500 pad - 0 to 46 checksum - 4

3.9.2 Minimum Packet (DATA) Size

46 bytes to ensure entire network is saturated so even computers at maximum distance away (300-500m) can detect errors.

data+pad >= 46

3.9.3 Max Packet Size

1500

3.9.4 Switched Ethernet

  • Early version (hub)
    Indistinquishable from a single wire. Just as many collisions as without.
  • Migration to switch
    Smarter than a hub, computers can just talk to the switch, and it can route that to the correct computer. When connections to the hub are duplex (two-way) there are no longer collisions because all computers are just talking to the switch.

3.12 Bluetooth

3.12.1 Transmission

Time-division duplex on ISM radio bandwidth (2.4Ghz). 79 separate channels (carrier frequencies) 1Mhz apart. Transmission stays on the same frequency until it has completed (but can only 1, 3, or 5 time slots)

3.12.2 Addresses

3.12.3 Active/Parked slaves

3.13 RFID

3.14 LAN Extension

3.14.1 Why?

Too many computers, wires too short, appearance of a logical LAN across large area

3.14.2 Repeater

Just connects segments, it's dumb and just retransmits everything it receives. Physical Layer

3.14.3 Bridge

Understands frames, it forwards frames that are not collisions but only if it needs to (it knows the recipient is on the other side). It keeps track of the hardware addresses, so simultaneous use of both segments can be used in parallel. Data Link Layer

3.14.4 Switch

Like a bridge between LANS. Data Link Layer

4 Network Layer

4.1 Purpose:

Get packets from source to destination. Uses hops (machine-to-machine) and the data link layer (machine to machine on wire). This layer needs to know about the whole network.

4.2 Store and Forward

  1. receive a packet
  2. find it's destination
  3. send it to the next router that is closer to the destination

4.3 Goals

Services should be independent of the routing of packets. Transport layer shouldn't have to deal with the network at all.

4.4 Connectionless Service

Uses datagrams; no two datagrams are related.

4.4.1 Packet Switching

Each switch knows enough about the network to be able to get any packet packet closer to its destination.

4.4.2 Failure

Router failure doesn't affect the packet transmission.

4.5 Connection-Oriented Service

4.5.1 Virtual Circuit

Once set up, must be notified by transport layer when to close the circuit. Each packet can just use a virtual circuit ID rather than a full address. BUT, every switch needs to know about the virtual circuit and reserve that slot.

4.5.2 Failuire

If a router fails, every single virtual circuit through that router dies.

4.6 Routing

Used mostly in WANs.

4.6.1 Types of switches

Interior switches usually don't have computers connected and are just used to route packets along. Exterior routers are usually connected to a LAN which transports packets to/from its computers.

4.6.2 Default Route

When a switch doesn't know where to send a packet, it should have a backup destination to forward it along

4.6.3 Routing Tables

Each switch can be assigned an identifier, and the switch "learns" what the shortest/fastest route to a certain switch.

4.7 Congestion Control

Mostly only cause of lost packets - most hardware is pretty solid, but can only handle congestion so well.

4.8 QoS

4.8.1 Jitter

Variation in delay. Delay is fine if it's consistent (in certain applications: video, audio) If we get a 1ms delay followed by a 1sec delay, we can't play video well (without a lot of buffering)

4.9 Internetworking

Connecting heterogeneous networks seamlessly to produce a virtual network. "Internet" is a nice clean virtual network that envelopes all these layer2 connected networks.

4.9.1 IP Addresses

prefix: network number. suffix: host-ID

  • Class A
    '0' prefix, followed by 7bit prefix, followed by 24bit suffix. 0 through 127 (128 nets, 16777216 hosts/net)
  • Class B
    '10' prefix, followed by 14bit prefix, followed by 16bit suffix. 128 through 191 (16384 nets, 65536 hosts/net)
  • Class C
    '110' prefix, followed by 21bit prefix, followed by 8bit suffix. 192 through 233 (2097152 nets, …)
  • Class D
    '1110' multicast address

4.10 Routers

Each LAN gets its own "unique" IP address

4.10.1 Mapping hardware address to protocol address

4.10.2 ARP

Address Resolution Protocol. Broadcast message requesting a hardware address for a peer. All listening computers can store the senders address. Response is directed, not broadcast. ONLY NEED HWADDR FOR LOCAL NETWORKS, just use proto address otherwise.

4.11 IP Datagrams

4.11.1 Virtual Packets

Makes packets identical on all technologies, giving a seamless interaction between different types of machines. Connectionless service, and is a "best-effort deliver" - no guarantees of dropped/duplicated/out of order/corrupted packets.

4.11.2 Fragmentation

Virtual IP packets an be huge (64k) but most data links have much smaller MTU (maximum transmission unit). Routers can/have to fragment IP packets in to smaller packets. These packets now flow through the net as individual packets, only to be reassembled at the final destination.

4.12 IPV6

128bits.

4.12.1 Extensible Headers

All headers put in the front of the virtual packet, so that additional headers can be created and used (extended..)

4.12.2 IPV6 Datagram

Just base header, extension headers, and data. No checksum.

4.12.3 Fragmentation

Routers don't fragment anymore, the sending host is responsible now.

4.12.4 Addressing

Uses hex; groups of 16bits. 2085:FFFF:0:0 … Double colon denotes all 0 groups between. FF0C::F1 is valid

4.12.5 6to4 net

2002:x:y/48 - xy is 32bit ipv4 address

4.13 UDP and TCP

Both transport layer protocols that use the network layer to send packets. End-to-end protocol.

4.13.1 Multiplexing

Allows unique process identification using ports numbers

4.13.2 Packets

TCP headers and data encapsulated by the IP protocol in the network layer.

4.14 NAT (network address translation)

Allows multiple computers to use one public IP address. To use this we need to set up a private IP address space.

4.15 ICMP (Internet Control Message Protocol)

Uses IP Packets; Reports errors, control issues.

4.15.1 Ping - echo/echo reply
